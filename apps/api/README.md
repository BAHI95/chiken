# Farm API

خادم الحسابات وحالة المزرعة مبني الآن على **Postgres** بدل SQLite وقت التشغيل، ويدعم:
- التحقق بالبريد الإلكتروني
- إعادة تعيين كلمة المرور
- مراقبة الصحة والجاهزية والـ metrics
- نسخ احتياطي JSON من البيانات الأساسية

الوظائف الأساسية:
- تسجيل حساب
- تسجيل دخول
- جلسات فعلية
- تخزين حالة كل مزرعة بشكل مستقل
- تشغيل migrations تلقائيًا عند إقلاع الخادم

## المتطلبات

- متغير بيئة `DATABASE_URL`
- قاعدة Postgres قابلة للاتصال من الخادم

## الأوامر المهمة

- تشغيل الخادم:
  - `npm run api:start`
- تشغيل migrations فقط:
  - `npm run api:migrate`
- ترحيل البيانات القديمة من SQLite إلى Postgres:
  - `npm run api:migrate:sqlite`
- أخذ نسخة احتياطية:
  - `npm run api:backup`
- استرجاع نسخة احتياطية:
  - `npm run api:restore -- <backup-file.json>`
- فحص الصحة:
  - `npm run api:health-check`

## مسار الترحيل الحالي

1. تشغيل migrations على Postgres
2. تشغيل سكربت `migrate:sqlite` مرة واحدة لنقل:
   - المستخدمين
   - حالة كل مزرعة
   - الجلسات
3. بعد التأكد من نجاح النقل، يصبح `farm-auth.db` مجرد مصدر قديم أو نسخة أرشيفية، وليس اعتمادًا وقت التشغيل
