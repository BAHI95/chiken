# تجهيز المشروع للرفع إلى GitHub

## قبل الرفع
- تنظيف الملفات المؤقتة والاختبارية.
- التأكد من أن النسخة الحالية تعمل محليًا.
- مراجعة الملفات الحساسة وعدم رفع أي أسرار مستقبلية.

## بعد الرفع
- تفعيل Issues للملاحظات.
- إنشاء Releases عند الوصول إلى نسخ مستقرة.
- إضافة لقطات شاشة داخل المستودع.
- إضافة خطة طريق للنسخة السحابية والحسابات.

## ما الذي نحتاجه لاحقًا
- تحديد هل البيع سيتم كمشروع خاص لعميل واحد أم كمنتج SaaS.
- إذا كان SaaS:
  - فرع إنتاج
  - فرع تطوير
  - CI بسيط
  - بيئة نشر
